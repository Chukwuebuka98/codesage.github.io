Python and JavaScript

While Python and JavaScript share some similarities, they also have fundamental differences. JavaScript, originally named LiveScript and later renamed JavaScript, shares some syntax and control-flow constructs with Java but does not have Java's static typing and strong type checking.

JavaScript uses a prototype-based object model instead of the class-based object model found in Java. It supports functions as methods and has a more flexible approach to variable declaration and typing.

Unlike Java, JavaScript does not require explicit declaration of variables, classes, or methods. Access modifiers like public, private, or protected are not needed, and interfaces are not mandatory. Variables, parameters, and function return types are not explicitly typed.

Variables

In Python, variables are used to store values. Variable names, called identifiers, must adhere to certain rules.

A Python identifier must start with a letter (a-z, A-Z) or an underscore (_). The subsequent characters can be letters, digits (0-9), or underscores. Python is case-sensitive, so uppercase and lowercase letters are distinct.

Examples of valid variable names: number_hits, temp99, _name.

Declaring variables
There are multiple ways to declare variables in Python:

Using the = assignment operator: x = 42 This syntax declares a variable and assigns it a value. It can be used for both local and global variables.

By directly assigning a value without using the var keyword: x = 42 This creates a global variable but is not recommended

Using the = assignment operator with the let keyword: y = 13 This syntax declares a block scope local variable. See Variable scope below.

Variable scope

When you declare a variable outside of any function, it is called a global variable, because it is available to any other code in the current document. When you declare a variable within a function, it is called a local variable, because it is available only within that function.

In Python, variable scope determines the accessibility and lifetime of a variable. Variables declared outside any function are global variables, accessible to all code within the current module. Variables declared within a function are local variables, only accessible within that function.

Unlike JavaScript, Python does not have block-level scope. In Python, a variable declared within a block (e.g., if statement) is still accessible outside the block if it was already declared.

if True: x = 5 print(x) # Output: 5

This behavior changes, when using the let declaration introduced in ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined
Constants

Python does not have built-in constants like JavaScript, but you can create named constants by convention using uppercase letters.

PI = 3.14

It's a common practice to treat these constants as read-only variables and not reassign them.

Data types

Python provides several built-in data types, including::

  • Numeric types:

    • Integer: whole numbers without a fractional part (e.g., 42).
    • Float: real numbers with a fractional part (e.g., 3.14159).
    • Boolean: True or False values representing logical truth values.
    • String: a sequence of characters enclosed in single quotes ('') or double quotes ("").
    • List: an ordered collection of items enclosed in square brackets ([]).
    • Dictionary: an unordered collection of key-value pairs enclosed in curly braces ({}) with a key-value mapping.
    • Tuple: an ordered, immutable collection of items enclosed in parentheses (()).
    • Set: an unordered collection of unique items enclosed in curly braces ({}) without duplicates.
Functions in Python
In Python, a function is defined using the def keyword followed by the function name, a list of arguments enclosed in parentheses, and a colon. The function body is indented and consists of statements.

Here's an example of a simple function named square that calculates the square of a number:

def square(number): return number * number The return statement specifies the value to be returned by the function. Function arguments in Python are passed by value.

Control structures: if...else statement and while statement Python provides the if...else statement for conditional execution and the while statement for repeated execution based on a condition:

The if...else statement syntax in Python is as follows:

if condition: statement_1 else: statement_2 The condition can be any expression that evaluates to either True or False. If the condition is True, statement_1 is executed; otherwise, statement_2 is executed.

You can also use elif (short for "else if") to test multiple conditions in sequence:

if condition_1: statement_1 elif condition_2: statement_2 elif condition_n: statement_n else: statement_last In Python, block statements are indicated by indentation, using spaces or tabs. It is recommended to use spaces for indentation.

The while statement is used to execute a block of statements as long as a specified condition is True. The syntax is as follows:

while condition: statement The condition is evaluated before each iteration. If the condition is True, the statement is executed, and the loop continues. When the condition becomes False, the loop terminates, and the program continues with the statement after the loop.

It's important to ensure that the condition eventually becomes False to prevent an infinite loop.

Function declarations
In Python, a function definition (or declaration) consists of the def keyword followed by the function name, a list of arguments enclosed in parentheses, and a colon. The function body, containing statements, is indented.

Here's an example of a function named square that calculates the square of a number:

def square(number): return number * number The return statement specifies the value to be returned by the function. Primitive parameters, such as numbers, are passed by value in Python, so changes within the function do not affect the original values.

It's worth noting that Python uses indentation to define blocks, so it's crucial to indent the function body correctly.
